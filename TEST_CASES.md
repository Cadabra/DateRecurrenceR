# DateRecurrenceR ‚Äî Public API Test Cases

A curated, ready-to-use bundle of behavioural test cases for the **public API** of
`DateRecurrenceR` (the `Recurrence` factory methods and the `Core` value types).

Every expected sequence in this document was **generated by executing the case against
the library** (net10.0) ‚Äî they are observed outputs, not hand-computed values. Use them
as a reference oracle for unit tests, as living documentation of the semantics, or as a
regression corpus.

- **Main** ‚Äî the common, happy-path behaviour a user relies on.
- **Corner** ‚Äî boundaries: `count = 0/1`, empty intersections, month/year clamping.
- **Tricky** ‚Äî leap days, `firstDayOfWeek` grouping, day-of-year drift, `Last` index,
  `DateOnly.MaxValue` limits, union de-duplication.

Dates are written `yyyy-MM-dd`. `‚àÖ` means the enumerator yields nothing.

---

## How to consume these cases from xUnit

All `Recurrence.*` methods return an `IEnumerator<DateOnly>` (or a concrete struct
enumerator that implements it). A single helper turns any case into a `DateOnly[]`:

```csharp
using DateRecurrenceR;
using DateRecurrenceR.Core;

internal static class RecurrenceTestHelper
{
    /// <summary>Drains an enumerator into an array (materialises the recurrence).</summary>
    public static DateOnly[] Collect(IEnumerator<DateOnly> e)
    {
        var result = new List<DateOnly>();
        while (e.MoveNext()) result.Add(e.Current);
        return result.ToArray();
    }

    /// <summary>Shorthand to build DateOnly[] from "yyyy-MM-dd" literals.</summary>
    public static DateOnly[] Dates(params string[] isoDates) =>
        isoDates.Select(DateOnly.Parse).ToArray();
}
```

A representative parametrised test (using xUnit + FluentAssertions, the stack this repo
already uses):

```csharp
public class DailyApiTests
{
    [Fact] // DAILY-M1 ¬∑ every day, count 5
    public void Daily_EveryDay_Count5()
    {
        var actual = RecurrenceTestHelper.Collect(
            Recurrence.Daily(new DateOnly(2024, 1, 1), count: 5, new Interval(1)));

        actual.Should().Equal(RecurrenceTestHelper.Dates(
            "2024-01-01", "2024-01-02", "2024-01-03", "2024-01-04", "2024-01-05"));
    }
}
```

Each row below gives you the exact call and the exact expected sequence, so it drops
straight into that pattern (or into a `[Theory]` / `[MemberData]` source).

## Subrange semantics (quick reference)

Every recurrence family supports two ways to bound the result:

| Overload shape | Meaning |
|---|---|
| `(beginDate, count, ‚Ä¶, interval)` | first `count` occurrences starting at/after `beginDate` |
| `(beginDate, fromDate, count, ‚Ä¶, interval)` | first `count` occurrences on/after `fromDate` |
| `(beginDate, endDate, ‚Ä¶, interval)` | occurrences in `[beginDate, endDate]` |
| `(beginDate, endDate, fromDate, toDate, ‚Ä¶, interval)` | occurrences in `[beginDate, endDate] ‚à© [fromDate, toDate]` |

`beginDate` always defines the **phase/anchor** of the pattern; `fromDate`/`toDate`
only **window** it ‚Äî they never shift the grid.

## 2. Weekly

`Recurrence.Weekly(beginDate, ‚Ä¶, WeekDays weekDays, DayOfWeek firstDayOfWeek, Interval)`

`WeekDays` can be built from explicit days, e.g.
`new WeekDays(DayOfWeek.Monday, DayOfWeek.Wednesday, DayOfWeek.Friday)`, or from seven
booleans `new WeekDays(isSunday, isMonday, ‚Ä¶, isSaturday)`.

Setup used below:
```csharp
var mwf     = new WeekDays(DayOfWeek.Monday, DayOfWeek.Wednesday, DayOfWeek.Friday);
var monOnly = new WeekDays(DayOfWeek.Monday);
var satSun  = new WeekDays(DayOfWeek.Saturday, DayOfWeek.Sunday);
```
> `2024-01-01` is a **Monday** ‚Äî handy anchor for weekly cases.

| ID | Type | Scenario | Call | Expected |
|----|------|----------|------|----------|
| WEEK-M1 | Main | Mon/Wed/Fri, count 6 | `Recurrence.Weekly(new DateOnly(2024,1,1), 6, mwf, DayOfWeek.Monday, new Interval(1))` | `2024-01-01, 2024-01-03, 2024-01-05, 2024-01-08, 2024-01-10, 2024-01-12` |
| WEEK-M2 | Main | Mondays, every 2nd week | `Recurrence.Weekly(new DateOnly(2024,1,1), 4, monOnly, DayOfWeek.Monday, new Interval(2))` | `2024-01-01, 2024-01-15, 2024-01-29, 2024-02-12` |
| WEEK-M3 | Main | Mon/Wed/Fri in a date range | `Recurrence.Weekly(new DateOnly(2024,1,1), new DateOnly(2024,1,14), mwf, DayOfWeek.Monday, new Interval(1))` | `2024-01-01, 2024-01-03, 2024-01-05, 2024-01-08, 2024-01-10, 2024-01-12` |
| WEEK-C1 | Corner | Begin on Tuesday ‚Üí first selected day is Wed | `Recurrence.Weekly(new DateOnly(2024,1,2), 4, mwf, DayOfWeek.Monday, new Interval(1))` | `2024-01-03, 2024-01-05, 2024-01-08, 2024-01-10` |

`Recurrence.Monthly(beginDate, ‚Ä¶, DayOfWeek dayOfWeek, IndexOfDay indexOfDay, Interval)`

## 5. Yearly ‚Äî by day of year

`Recurrence.Yearly(beginDate, ‚Ä¶, DayOfYear dayOfYear, Interval)`

`DayOfYear` accepts `1..366`. Day 366 in a non-leap year **clamps to Dec 31**. Note that
a fixed day-of-year maps to a **different calendar date** across leap boundaries.

| ID | Type | Scenario | Call | Expected |
|----|------|----------|------|----------|
| YRDOY-M1 | Main | Day 100 each year (drifts around Apr) | `Recurrence.Yearly(new DateOnly(2024,1,1), 3, new DayOfYear(100), new Interval(1))` | `2024-04-09, 2025-04-10, 2026-04-10` |
| YRDOY-M2 | Tricky | Day 60 ‚Üí Feb 29 in leap, Mar 1 otherwise | `Recurrence.Yearly(new DateOnly(2024,1,1), 3, new DayOfYear(60), new Interval(1))` | `2024-02-29, 2025-03-01, 2026-03-01` |
| YRDOY-T1 | Tricky | Day 366 clamps to Dec 31 in a non-leap year | `Recurrence.Yearly(new DateOnly(2023,1,1), 2, new DayOfYear(366), new Interval(1))` | `2023-12-31, 2024-12-31` |
| YRDOY-M3 | Main | Day 200, every 2 years, in a range | `Recurrence.Yearly(new DateOnly(2024,1,1), new DateOnly(2028,12,31), new DayOfYear(200), new Interval(2))` | `2024-07-18, 2026-07-19, 2028-07-18` |

> **YRDOY-M1** is the canonical "day-of-year drift" assertion: 2024 is a leap year so day
> 100 = Apr 9, but in common years day 100 = Apr 10. A good guard for leap-year
> off-by-one regressions.

## 8. Union

`Recurrence.Union(e1, e2)`, `Recurrence.Union(e1, e2, e3)`,
`Recurrence.Union(params IEnumerator<DateOnly>[])`, and an `IEnumerable<‚Ä¶>` overload.
Produces the **sorted, de-duplicated** merge of ascending inputs.

| ID | Type | Scenario | Call | Expected |
|----|------|----------|------|----------|
| UNION-M1 | Main | Merge "every 2 days" and "every 3 days" | see below | `2024-01-01, 2024-01-03, 2024-01-04, 2024-01-05, 2024-01-07, 2024-01-09, 2024-01-10` |
| UNION-T1 | Tricky | Overlapping ranges de-duplicate | see below | `2024-01-01, 2024-01-02, 2024-01-03, 2024-01-04` |
| UNION-T2 | Tricky | Three-way union of different families | see below | `2024-01-01, 2024-01-08, 2024-01-15, 2024-01-22, 2024-01-29` |

```csharp
// UNION-M1
Recurrence.Union(
    Recurrence.Daily(new DateOnly(2024,1,1), new DateOnly(2024,1,10), new Interval(2)),   // 1,3,5,7,9
    Recurrence.Daily(new DateOnly(2024,1,1), new DateOnly(2024,1,10), new Interval(3)));  // 1,4,7,10

// UNION-T1 ‚Äî the two streams share 2 and 3; union keeps each once
Recurrence.Union(
    Recurrence.Daily(new DateOnly(2024,1,1), 3, new Interval(1)),                         // 1,2,3
    Recurrence.Daily(new DateOnly(2024,1,2), 3, new Interval(1)));                        // 2,3,4

// UNION-T2 ‚Äî Mondays ‚à™ the 1st of the month ‚à™ a single day (Jan 15)
Recurrence.Union(
    Recurrence.Weekly(new DateOnly(2024,1,1), new DateOnly(2024,1,31), new WeekDays(DayOfWeek.Monday), DayOfWeek.Monday, new Interval(1)),
    Recurrence.Monthly(new DateOnly(2024,1,1), new DateOnly(2024,1,31), new DayOfMonth(1), new Interval(1)),
    Recurrence.Daily(new DateOnly(2024,1,15), 1, new Interval(1)));
```

> **Contract:** `Union` assumes each input enumerator is already ascending. Jan 2024's
> Mondays are 1, 8, 15, 22, 29; adding "the 1st" (Jan 1, duplicate) and the single day
> Jan 15 (duplicate) changes nothing ‚Äî that's the de-dup assertion in UNION-T2.

## 9. Argument validation (value types)

These guard the constructors of the `Core` value types and `Range`. They throw at
construction time, before any recurrence is produced.

| ID | Type | Construction | Expected |
|----|------|--------------|----------|
| VAL-INT-0   | Corner | `new Interval(0)` | throws `ArgumentOutOfRangeException` |
| VAL-DOM-0   | Corner | `new DayOfMonth(0)` | throws `ArgumentOutOfRangeException` |
| VAL-DOM-32  | Corner | `new DayOfMonth(32)` | throws `ArgumentOutOfRangeException` |
| VAL-DOY-0   | Corner | `new DayOfYear(0)` | throws `ArgumentOutOfRangeException` |
| VAL-DOY-367 | Corner | `new DayOfYear(367)` | throws `ArgumentOutOfRangeException` |
| VAL-MOY-0   | Corner | `new MonthOfYear(0)` | throws `ArgumentOutOfRangeException` |
| VAL-MOY-13  | Corner | `new MonthOfYear(13)` | throws `ArgumentOutOfRangeException` |
| VAL-RANGE-NEG | Corner | `new DateRange(new DateOnly(2024,1,1), count: -1)` | throws `ArgumentException` |

Valid boundary values (should **not** throw): `new Interval(1)`, `new DayOfMonth(1)`,
`new DayOfMonth(31)`, `new DayOfYear(1)`, `new DayOfYear(366)`, `new MonthOfYear(1)`,
`new MonthOfYear(12)`. Parameterless constructors yield the minimum value
(`Interval() == Interval.MinValue == 1`, etc.).

Example:
```csharp
[Fact] // VAL-INT-0
public void Interval_Zero_Throws() =>
    ((Action)(() => new Interval(0))).Should().Throw<ArgumentOutOfRangeException>();
```
